1 NIVEAU 1 — RÉSEAU ROUTIER
Modéliser la ville comme un graphe
On veut trouver le chemin le plus court entre n'importe quels deux points de collecte. Pour ça, on modélise le réseau routier comme un graphe pondéré et on utilise l'algorithme de Dijkstra.
🎯 Objectif : Calculer la matrice 10×10 des distances optimales entre tous les points (dépôt + 9 quartiers).
1

Qu'est-ce qu'un graphe ?

La structure de données de base du projet

Sommet (Nœud)

Un point dans le graphe. Ici : le dépôt (id=0) ou un quartier de collecte.

Arête

Une connexion entre deux sommets. Ici : une route qui relie deux points de collecte.

Poids

La valeur associée à une arête. Ici : la distance en km entre deux points.

Exemple — graphe à 5 nœuds
ℹ️Dans notre projet, le graphe est non-orienté (on peut circuler dans les deux sens sur chaque route) et connexe (tous les points sont accessibles depuis le dépôt).
2

Modéliser la ville en Python

Deux classes : PointCollecte et GrapheRoutier

point_collecte.py
class PointCollecte:
    def __init__(self, id_point, x, y, nom=""):
        self.id  = id_point   # ex: 0 = dépôt, 1 = Quartier Nord
        self.x   = x          # coordonnée X sur la carte
        self.y   = y          # coordonnée Y sur la carte
        self.nom = nom        # nom lisible (optionnel)

    def distance_vers(self, autre):
        # Formule : √((x2-x1)² + (y2-y1)²)
        import math
        return math.sqrt((autre.x - self.x)**2 + (autre.y - self.y)**2)
graphe_routier.py
class GrapheRoutier:
    def __init__(self):
        self.sommets    = {}  # { id → PointCollecte }
        self.aretes     = {}  # { (id1,id2) → distance }
        self._adjacence = {}  # { id → [(voisin, distance)] }

    def ajouter_arete(self, id1, id2, distance=None):
        if distance is None:
            distance = self.sommets[id1].distance_vers(self.sommets[id2])
        # Non-orienté → stocker les 2 sens
        self._adjacence[id1].append((id2, distance))
        self._adjacence[id2].append((id1, distance))

graphe_routier.py — plus_court_chemin()
import heapq

def plus_court_chemin(self, depart, arrivee):
    distances     = {id: float('inf') for id in self.sommets}
    predecesseurs = {id: None for id in self.sommets}
    distances[depart] = 0
    file = [(0, depart)]  # (distance, sommet)
    visites = set()

    while file:
        dist_actuelle, sommet = heapq.heappop(file)  # extraire le + proche
        if sommet in visites: continue
        visites.add(sommet)

        for voisin, poids in self._adjacence[sommet]:
            nouvelle_dist = dist_actuelle + poids
            if nouvelle_dist < distances[voisin]:   # amélioration trouvée ?
                distances[voisin]     = nouvelle_dist
                predecesseurs[voisin] = sommet
                heapq.heappush(file, (nouvelle_dist, voisin))

    # Reconstruire le chemin en remontant les prédécesseurs
    chemin, cur = [], arrivee
    while cur is not None:
        chemin.append(cur)
        cur = predecesseurs[cur]
    return distances[arrivee], chemin[::-1]
❓ Pourquoi utilise-t-on un tas (heapq) plutôt qu'une liste simple ?
🅐 Pour trier les nœuds par nom alphabétique
🅑 Pour extraire le nœud de distance minimale en O(log V) au lieu de O(V)
🅒 Pour éviter les doublons dans le graphe
🅓 Pour stocker les arêtes plus efficacement
✅ Exact ! La file de priorité min (tas binaire) permet d'extraire le minimum en O(log V). Sans ça, chercher le minimum dans une liste coûte O(V), ce qui porte la complexité totale à O(V²) au lieu de O((V+E)logV).

Matrice des distances

On lance Dijkstra depuis chaque sommet pour remplir toute la matrice

✅3 propriétés à vérifier : (1) Diagonale = 0, (2) Symétrie d(i,j)=d(j,i), (3) Inégalité triangulaire d(i,k) ≤ d(i,j)+d(j,k).

graphe_routier.py — matrice_distances()
def matrice_distances(self):
    ids = sorted(self.sommets.keys())
    matrice = {}
    for i in ids:
        matrice[i] = {}
        for j in ids:
            if i == j:
                matrice[i][j] = 0.0
            else:
                dist, _ = self.plus_court_chemin(i, j)  # Dijkstra
                matrice[i][j] = dist
    return matrice  # Complexité totale : O(V × (V+E)logV)

Graphe biparti — Définition

Deux groupes distincts : les arêtes ne connectent que des nœuds de groupes différents

🔑Un graphe G=(C∪Z, E) est biparti si ses sommets se divisent en 2 ensembles C et Z tels qu'aucune arête ne relie deux sommets du même ensemble. Ici C = camions, Z = zones.

affectateur_biparti.py — affectation_gloutonne()
def affectation_gloutonne(self):
    # ÉTAPE 1 : trier les zones par volume décroissant
    zones_triees = sorted(self.zones.values(),
                           key=lambda z: z.volume_moyen, reverse=True)
    affectation = {cid: [] for cid in self.camions}

    for zone in zones_triees:
        meilleur_camion, meilleur_cout = None, float('inf')

        # ÉTAPE 2 : trouver le camion le moins chargé accessible
        for camion in self.camions.values():
            if not camion.peut_servir_zone(zone.id): continue
            nouvelle_charge = camion.charge_actuelle + zone.volume_moyen
            if nouvelle_charge > camion.capacite: continue  # trop lourd
            cout = camion.charge_actuelle  # critère : charge actuelle
            if cout < meilleur_cout:
                meilleur_cout, meilleur_camion = cout, camion

        # ÉTAPE 3 : affecter et mettre à jour la charge
        if meilleur_camion:
            affectation[meilleur_camion.id].append(zone.id)
            meilleur_camion.charge_actuelle += zone.volume_moyen
    return affectation

3 NIVEAU 3 — CRÉNEAUX TEMPORELS
Quand collecter ? Camion ↔ Zone ↔ Créneau
On ajoute la dimension temps. Chaque zone n'est accessible qu'à certaines heures. Un camion ne peut pas être à deux endroits simultanément. Le modèle devient un graphe triparti.
🎯 Objectif : Générer un planning hebdomadaire sans conflits horaires, 0 chevauchement.
1

Graphe triparti — 3 ensembles de nœuds

On étend le graphe biparti avec un troisième ensemble : les créneaux

C — Camions

3 camions. Chacun a des zones accessibles et des pauses obligatoires (ex: déjeuner 12h-13h).

Z — Zones

6 zones avec une fenêtre horaire d'accessibilité : ex. zone résidentielle interdite la nuit.

T — Créneaux

9 créneaux de 2h sur 5 jours. Chaque créneau a un coefficient de congestion (1.0 à 2.0).

🔑Un triplet (ci, zj, tk) est réalisable si ET seulement si : (1) ci accède à zj, (2) zj ouverte pendant tk, (3) ci pas en pause pendant tk.

Fenêtres horaires

Chaque zone a une plage d'accessibilité — en dehors, la collecte est interdite

def est_realisable(self, camion_id, zone_id, creneau):
    # 1. Vérifier la fenêtre horaire de la zone
    if zone_id in self.fenetres_zone:
        debut_ok, fin_ok = self.fenetres_zone[zone_id]
        if creneau.debut < debut_ok or creneau.fin > fin_ok:
            return False  # hors fenêtre → refusé

    # 2. Vérifier les pauses du camion
    for p_debut, p_fin in self.pauses_camion.get(camion_id, []):
        # Chevauchement : debut1 < fin2 ET debut2 < fin1
        if creneau.debut < p_fin and p_debut < creneau.fin:
            return False  # conflit avec pause → refusé

    return True  # toutes les contraintes OK

4 NIVEAU 4 — VRP
Quel trajet exact ? Vehicle Routing Problem
On génère les itinéraires concrets de chaque camion dans quel ordre visiter les points. C'est un problème NP-difficile : on utilise des heuristiques en 3 phases.
🎯 Objectif : Réduire la distance de 50.63 km à 38.39 km (−24.2%) via construction + 2-opt + Tabou.
1

Problème VRP — Formulation mathématique

NP-difficile : pas de solution exacte en temps polynomial pour de grandes instances

Minimiser : ∑k ∑(i,j) dij · xijk
xijk=1 si camion k va de i vers j · Contrainte capacité · Contrainte unicité

Phase 1 — Construction

Plus proche voisin : greedy O(n²). Rapide, solution de base raisonnable.

Phase 2 — 2-opt

Amélioration locale : inverse des segments pour supprimer les croisements.

Phase 3 — Tabou

Métaheuristique globale : explore hors du minimum local via liste tabou.

2

Plus Proche Voisin — Construction

Depuis le dépôt, on visite toujours le point non-visité le plus proche

Simulation interactive — cliquer Suivant
Étape 0 / 9
On part du dépôt (point 0). À chaque étape, on va au point le plus proche non encore visité.
optimiseur_vrp.py — construire_solution_initiale()
def construire_solution_initiale(self):
    non_visites = set(range(1, len(self.points)))  # sans le dépôt (0)
    tournee = [0]  # commencer au dépôt
    position = 0

    while non_visites:
        # Chercher le voisin le plus proche non-visité
        plus_proche = min(non_visites,
            key=lambda p: self.matrice[position][p])

        # Vérifier la capacité avant d'ajouter
        charge_future = sum(self.demandes[p] for p in tournee[1:]) + self.demandes[plus_proche]
        if charge_future > self.capacite_camion:
            break  # camion plein → passer au suivant

        tournee.append(plus_proche)
        non_visites.discard(plus_proche)
        position = plus_proche

    tournee.append(0)  # retour au dépôt
    return tournee

Algorithme 2-opt — Amélioration locale

Inverse des segments pour éliminer les croisements dans la tournée

✅Intuition : Si deux arêtes de la tournée se croisent, les supprimer et reconnecter autrement réduit toujours la distance. Le 2-opt élimine tous les croisements.
Démonstration 2-opt — Avant / Après
Tournée initiale avec un croisement. Cliquer pour voir l'amélioration 2-opt.
optimiseur_vrp.py — algorithme_2opt()
def algorithme_2opt(self, tournee):
    ameliore = True
    while ameliore:          # continuer tant qu'il y a des améliorations
        ameliore = False
        n = len(tournee)
        for i in range(1, n - 2):
            for j in range(i + 1, n - 1):
                # Tester : vaut-il mieux inverser le segment [i..j] ?
                d_avant = self.matrice[tournee[i-1]][tournee[i]] \
                        + self.matrice[tournee[j]][tournee[j+1]]
                d_apres = self.matrice[tournee[i-1]][tournee[j]] \
                        + self.matrice[tournee[i]][tournee[j+1]]
                if d_apres < d_avant - 1e-10:  # amélioration trouvée
                    # Inverser le segment
                    tournee[i:j+1] = tournee[i:j+1][::-1]
                    ameliore = True
    return tournee

Recherche Tabou — Optimisation globale

Accepte des solutions légèrement moins bonnes pour échapper aux minima locaux

Paysage de solutions — Pourquoi la Tabou ?

Liste Tabou

Mémorise les 10 derniers mouvements. On ne peut pas les refaire — évite les cycles.

Mouvement

Échanger un point entre deux tournées. On teste plusieurs candidats par itération.

Arrêt

100 itérations max, ou 15 itérations sans amélioration (stagnation détectée).

❓ Pourquoi la Recherche Tabou est-elle meilleure que le 2-opt seul ?
🅐 Parce qu'elle est toujours plus rapide
🅑 Parce qu'elle peut accepter des solutions pires temporairement pour sortir des minima locaux
🅒 Parce qu'elle garantit l'optimum global
🅓 Parce qu'elle utilise moins de mémoire
✅ Exact ! Le 2-opt s'arrête dès qu'il n'y a plus d'amélioration locale. La Tabou peut "remonter" la colline pour explorer de nouveaux voisinages et atteindre de meilleures solutions globales. Résultat : −24.2% de distance sur notre instance.

class CapteurIoT:
    def __init__(self, point_id, niveau_initial=0.0):
        self.niveau        = niveau_initial
        self.seuil_alerte  = 80.0   # % → notifier opérateur
        self.seuil_urgence = 90.0   # % → replanification immédiate
        self._taux_h       = random.uniform(2, 8)  # %/heure
        self.historique    = []

    def simuler_remplissage(self, heures):
        bruit = random.gauss(0, 0.5)  # bruit réaliste
        self.niveau = min(100.0, self.niveau + self._taux_h * heures + bruit)
        self.historique.append((heures, self.niveau))

    def detecter_urgence(self):
        return self.niveau >= self.seuil_urgence  # True → replanifier !

Dominance de Pareto — Définition précise

La règle pour identifier les meilleures solutions du front

A domine B si : A ≤ B sur TOUS les critères ET A < B sur AU MOINS UN critère
Exemple concret — 4 solutions, 2 objectifs à minimiser
systeme_dynamique.py — dominance + front_pareto()
def domine(scores_a, scores_b, types):
    """True si A domine B. types = liste de 'min'/'max' par objectif."""
    meilleur_partout    = True
    meilleur_quelquepart = False
    for a, b, t in zip(scores_a, scores_b, types):
        if t == 'min':
            if a > b: meilleur_partout = False
            if a < b: meilleur_quelquepart = True
        else:  # 'max'
            if a < b: meilleur_partout = False
            if a > b: meilleur_quelquepart = True
    return meilleur_partout and meilleur_quelquepart

def trouver_front_pareto(solutions, types):
    n = len(solutions)
    est_domine = [False] * n
    for i in range(n):
        for j in range(n):
            if i != j and domine(solutions[j], solutions[i], types):
                est_domine[i] = True
                break
    return [s for s, d in zip(solutions, est_domine) if not d]
❓ Peut-il y avoir UNE SEULE solution sur le front de Pareto ?
🅐 Oui, si une solution est meilleure sur tous les critères simultanément
🅑 Non, le front de Pareto contient toujours plusieurs solutions
🅒 Non, il faut au minimum autant de solutions que d'objectifs
✅ Exact ! Si une solution domine toutes les autres sur tous les critères, elle est la seule non-dominée. En pratique, avec des objectifs vraiment contradictoires (coût vs CO2 vs satisfaction), on trouve généralement plusieurs solutions sur le front.

import math

def distance_euclidienne(x1, y1, x2, y2) -> float:
    return math.sqrt((x2 - x1)**2 + (y2 - y1)**2)

def ecart_type(valeurs: list) -> float:
    n = len(valeurs)
    if n == 0: return 0.0
    mu = sum(valeurs) / n
    return math.sqrt(sum((v - mu)**2 for v in valeurs) / n)

def distance_route(tournee: list, matrice: list) -> float:
    """Calcule distance totale d'une tournée (liste d'indices)."""

               for i in range(len(tournee) - 1))

ContrainteTemporelle — calculer_penalite()

N3 : calcule la pénalité en €/min quand un camion est en retard ou dans un créneau chargé

Pénalité retard

Si le créneau est dépassé : chaque minute de retard coûte une pénalité fixe (ex: 5 €/min).

Pénalité congestion

Si le niveau de congestion est > 1.0, le coût est multiplié : tarif × congestion.

Pénalité hors fenêtre

Si une zone résidentielle est visitée la nuit → pénalité max (plainte citoyenne).

pénalité = retard_min × 5.0 + (congestion − 1.0) × 20.0 + hors_fenetre × 100.0
contrainte_temporelle.py — calculer_penalite()
def calculer_penalite(self, camion_id: int, zone_id: int, creneau) -> float:
    """
    Calcule la pénalité temporelle pour un triplet (camion, zone, créneau).
    Retourne un float ≥ 0. Plus c'est élevé, moins l'affectation est souhaitable.
    """
    penalite = 0.0

    # ── 1. Pénalité de congestion ───────────────────────────────────
    # Si la congestion est élevée, le trajet prend plus longtemps → surcoût
    congestion = self.congestion.get((zone_id, creneau.id), 1.0)
    if congestion > 1.0:
        penalite += (congestion - 1.0) * 20.0  # 20€ par niveau de congestion

    # ── 2. Pénalité hors fenêtre horaire ────────────────────────────
    if zone_id in self.fenetres_zone:
        debut_autorise, fin_autorise = self.fenetres_zone[zone_id]
        if creneau.debut < debut_autorise:
            retard = (debut_autorise - creneau.debut).seconds / 60
            penalite += retard * 5.0    # 5€ par minute en dehors de la fenêtre
        elif creneau.fin > fin_autorise:
            retard = (creneau.fin - fin_autorise).seconds / 60
            penalite += retard * 5.0

    # ── 3. Pénalité zone résidentielle la nuit ──────────────────────
    # Zones interdites de nuit (22h-6h) → plainte citoyenne garantie
    ZONES_NUIT = getattr(self, 'zones_interdites_nuit', [])
    if zone_id in ZONES_NUIT:
        heure_fin = creneau.fin.hour
        if heure_fin >= 22 or heure_fin < 6:
            penalite += 100.0  # pénalité maximale

    # ── 4. Pénalité pause manquée ───────────────────────────────────
    for p_debut, p_fin in self.pauses_camion.get(camion_id, []):
        if creneau.debut < p_fin and p_debut < creneau.fin:
            penalite += 50.0  # conflit avec pause réglementaire

    return penalite

Classe Tournee — ajouter_point() & verifier_faisabilite()

N4 : gérer l'ordre des points et valider toutes les contraintes opérationnelles

tournee.py — ajouter_point() + verifier_faisabilite()
class Tournee:
    def __init__(self, camion_id: int, capacite_max: float):
        self.camion_id     = camion_id
        self.capacite_max  = capacite_max
        self.points        = []      # liste ordonnée d'IDs (commence et finit au dépôt 0)
        self.charge_actuelle = 0.0
        self.heure_depart  = None
        self.heure_retour  = None

    def ajouter_point(self, point_id: int, demande: float,
                       position: int = -1) -> bool:
        """
        Insère un point dans la tournée.
        position=-1 → à la fin (avant le retour dépôt)
        Retourne False si la capacité serait dépassée.
        """
        # Vérification capacité AVANT d'insérer
        if self.charge_actuelle + demande > self.capacite_max:
            return False  # refus : camion trop plein

        if position == -1:
            self.points.append(point_id)
        else:
            self.points.insert(position, point_id)

        self.charge_actuelle += demande
        return True  # succès

    def calculer_distance(self, matrice: list) -> float:
        """Somme des distances entre points consécutifs (dépôt → ... → dépôt)."""
        route = [0] + self.points + [0]  # dépôt au début et à la fin
        return sum(matrice[route[i]][route[i+1]] for i in range(len(route)-1))

    def calculer_duree(self, temps_service: dict, vitesse_kmh: float = 30) -> float:
        """
        Durée totale = temps trajet + temps de service à chaque point.
        temps_service : dict { point_id : minutes }
        """
        route = [0] + self.points + [0]
        duree = 0.0
        for i in range(len(route) - 1):
            # temps trajet entre deux points (en minutes)
            dist = matrice[route[i]][route[i+1]]
            duree += (dist / vitesse_kmh) * 60
        for pt in self.points:
            duree += temps_service.get(pt, 0)  # temps de vidage du conteneur
        return duree

    def verifier_faisabilite(self, contraintes: dict) -> bool:
        """
        Vérifie TOUTES les contraintes opérationnelles.
        contraintes = { 'capacite_max': float, 'fenetre_fin': str,
                        'demandes': dict, 'acces': dict }
        Retourne False dès qu'une contrainte est violée.
        """
        # 1. Contrainte capacité
        if self.charge_actuelle > contraintes.get('capacite_max', float('inf')):
            return False

        # 2. Contrainte fenêtre horaire globale
        if self.heure_retour and 'fenetre_fin' in contraintes:
            if self.heure_retour > contraintes['fenetre_fin']:
                return False  # retour trop tardif

        # 3. Contrainte d'accès pour chaque point
        acces = contraintes.get('acces', {})
        for pt in self.points:
            type_acces = acces.get(pt, 'facile')
            if type_acces == 'camion_mini' and self.capacite_max > 3000:
                return False  # trop grand pour cette rue étroite

        return True  # toutes les contraintes respectées

OptimiseurVRP — recherche_tabou() complète

N4 : implémentation complète avec liste tabou, critère d'aspiration et arrêt par stagnation

Mouvement

Déplacer un point d'une tournée vers une autre. Le mouvement est mémorisé dans la liste tabou.

Liste tabou

Les 10 derniers mouvements sont interdits. Évite de revenir en arrière et de boucler.

Aspiration

Exception : si un mouvement tabou donne la meilleure solution globale jamais vue, on l'accepte quand même.

optimiseur_vrp.py — recherche_tabou()
def recherche_tabou(self, iterations: int = 100,
                       taille_tabou: int = 10,
                       stagnation_max: int = 15) -> list:
    """
    Optimisation globale par Recherche Tabou.
    Retourne la meilleure liste de Tournee trouvée.
    """
    # ── Initialisation ──────────────────────────────────────────────
    tournees_actuelles = self.construire_solution_initiale()
    # Appliquer 2-opt sur chaque tournée pour partir d'une bonne base
    tournees_actuelles = [self.algorithme_2opt(t) for t in tournees_actuelles]

    meilleure_solution  = [t.copie() for t in tournees_actuelles]
    meilleur_cout       = self._cout_total(meilleure_solution)
    liste_tabou         = []   # [(tournee_src, tournee_dst, point_id)]
    stagnation          = 0

    for it in range(iterations):
        # ── Critère d'arrêt anticipé ────────────────────────────────
        if stagnation >= stagnation_max:
            break

        meilleur_voisin = None
        meilleur_delta  = float('inf')
        meilleur_mouvement = None

        # ── Explorer les voisins (déplacer un point entre tournées) ──
        for i_src, t_src in enumerate(tournees_actuelles):
            for point_id in t_src.points:
                for i_dst, t_dst in enumerate(tournees_actuelles):
                    if i_src == i_dst: continue

                    mouvement = (i_src, i_dst, point_id)
                    est_tabou = mouvement in liste_tabou

                    # Simuler le mouvement
                    delta = self._evaluer_mouvement(tournees_actuelles,
                                                    i_src, i_dst, point_id)

                    cout_voisin = self._cout_total(tournees_actuelles) + delta

                    # ── Critère d'aspiration : accepter si meilleur global ──
                    aspiration_ok = cout_voisin < meilleur_cout

                    if (not est_tabou or aspiration_ok) and delta < meilleur_delta:
                        meilleur_delta     = delta
                        meilleur_mouvement = mouvement
                        meilleur_voisin    = (i_src, i_dst, point_id)

        if meilleur_voisin is None:
            break  # aucun mouvement possible

        # ── Appliquer le meilleur mouvement ─────────────────────────
        i_src, i_dst, point_id = meilleur_voisin
        self._appliquer_mouvement(tournees_actuelles, i_src, i_dst, point_id)

        # ── Mettre à jour liste tabou (FIFO, taille limitée) ────────
        liste_tabou.append(meilleur_mouvement)
        if len(liste_tabou) > taille_tabou:
            liste_tabou.pop(0)  # supprimer le plus vieux

        # ── Mise à jour de la meilleure solution globale ─────────────
        cout_actuel = self._cout_total(tournees_actuelles)
        if cout_actuel < meilleur_cout:
            meilleur_cout      = cout_actuel
            meilleure_solution = [t.copie() for t in tournees_actuelles]
            stagnation         = 0   # amélioration → remettre compteur à 0
        else:
            stagnation += 1         # pas d'amélioration → incrémenter stagnation

    return meilleure_solution
✅Critère d'arrêt : 100 itérations max OU 15 itérations sans amélioration (stagnation). Le deuxième est souvent déclenché en premier, ce qui accélère l'exécution.

simulateur_temps_reel.py — replanifier_urgence()
def replanifier_urgence(self, points_urgents: list) -> list:
    """
    Insère les points urgents en tête des tournées en cours.
    Stratégie : trouver le camion le plus proche de chaque point urgent.
    Doit s'exécuter en < 2 secondes (pas de Tabou, insertion directe).
    """
    import time
    debut = time.time()

    for point_urgent_id in points_urgents:
        # Trouver le camion le plus proche du point urgent
        meilleur_camion, distance_min = None, float('inf')

        for tournee in self.tournees_en_cours:
            # Position actuelle = dernier point visité de la tournée
            pos_actuelle = tournee.position_actuelle if tournee.points else 0
            dist = self.graphe.matrice[pos_actuelle][point_urgent_id]

            if dist < distance_min:
                distance_min  = dist
                meilleur_camion = tournee

        # Insérer en PREMIÈRE position de la tournée restante
        if meilleur_camion is not None:
            meilleur_camion.points.insert(0, point_urgent_id)
            self.evenements.append({
                'type': 'replanification',
                'point': point_urgent_id,
                'camion': meilleur_camion.camion_id,
                'temps_reponse_s': round(time.time() - debut, 3)
            })

    return self.tournees_en_cours

class DashboardTempsReel:
    def __init__(self, simulateur):
        self.simulateur = simulateur
        self.metrics = {
            'co2_total': 0.0, 'cout_total': 0.0,
            'satisfaction': 100.0, 'points_urgents': []
        }

    def mettre_a_jour(self) -> None:
        """Recalcule tous les indicateurs à partir de l'état courant."""
        tournees = self.simulateur.tournees_en_cours
        capteurs  = self.simulateur.capteurs

        # CO2 accumulé (0.3 kg / km parcouru)
        dist_totale = sum(t.calculer_distance(self.simulateur.graphe.matrice)
                        for t in tournees)
        self.metrics['co2_total'] += dist_totale * 0.3

        # Coût cumulé (carburant + fixe)
        self.metrics['cout_total'] += dist_totale * 0.5

        # Points urgents (niveau > 90%)
        self.metrics['points_urgents'] = [
            c.point_id for c in capteurs.values() if c.detecter_urgence()
        ]

        # Satisfaction : baisse si retards ou urgences non traitées
        nb_urgents = len(self.metrics['points_urgents'])
        self.metrics['satisfaction'] = max(0, 100 - nb_urgents * 5)

    def generer_rapport(self, periode: str = "journalier") -> dict:
        """Génère un rapport récapitulatif de la période."""
        return {
            'periode': periode,
            'co2_total_kg': round(self.metrics['co2_total'], 2),
            'cout_total_euros': round(self.metrics['cout_total'], 2),
            'satisfaction_moyenne': self.metrics['satisfaction'],
            'points_urgents_traites': len(self.metrics['points_urgents']),
            'evenements': self.simulateur.evenements
        }

    def alerter_operateur(self, type_alerte: str, severite: int) -> None:
        """
        Envoie une alerte (console / log / notification).
        severite : 1=info, 2=avertissement, 3=critique
        """

        msg = f"[{niveau_txt}] {type_alerte} — heure: {self.simulateur.heure_actuelle}"
        print(msg)  # remplacer par logger.warning(msg) en production

        if severite == 3:  # critique → déclencher replanification immédiate
            urgents = self.metrics['points_urgents']
            if urgents:
                self.simulateur.replanifier_urgence(urgents)
