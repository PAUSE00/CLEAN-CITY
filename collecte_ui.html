Graphe Routier — Dijkstra

Modélise la ville comme un graphe pondéré. Calcule les distances optimales entre tous les points via l'algorithme de Dijkstra O((V+E)logV).

2

Graphe Biparti — Affectation Gloutonne

Assigne optimalement les zones aux camions selon capacité et accessibilité. Équilibrage itératif des charges.

3

Graphe Triparti — Contraintes Temporelles

Ajoute la dimension temps : fenêtres horaires, pauses obligatoires, congestion. Planification hebdomadaire.

4

VRP — 2-opt + Recherche Tabou

Génère les trajets concrets. Construction par plus-proche-voisin, amélioration 2-opt, optimisation globale Tabou.

5

Système Dynamique — NSGA-II + IoT

Simulation temps réel : capteurs IoT, événements aléatoires, multi-objectif Pareto. Replanification automatique.

Algorithme Dijkstra — Étapes
1

Initialisation

dist[source]=0, dist[tous les autres]=∞. File de priorité avec (0, source).

2

Exploration

Extraire le sommet u de distance minimale. Pour chaque voisin v : si dist[u]+w(u,v) < dist[v] → mettre à jour.

3

Reconstruction

Remonter les prédécesseurs de l'arrivée vers le départ pour retrouver le chemin.
